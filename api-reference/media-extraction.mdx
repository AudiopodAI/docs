## Overview

The AudioPod AI Media Extraction API enables seamless download and processing of audio content from various online sources including YouTube, social media platforms, and direct media URLs. Our advanced system supports direct YouTube integration for processing videos, playlists, and live streams, with automatic format detection and quality optimization.

<Info>
  Our media extraction service respects platform terms of service and copyright
  laws. Only extract content you have rights to use or that falls under fair use
  provisions.
</Info>

## Quick Start

### Extract from YouTube

<Tabs>
  <Tab title="POST">
    ```http
    POST /media/extract
    Authorization: Bearer {access_token}
    Content-Type: application/json
    
    {
      "url": "https://youtube.com/watch?v=dQw4w9WgXcQ",
      "format": "mp3",
      "quality": "high"
    }
    ```
  </Tab>
  <Tab title="Python">
    ```python
    import requests
    
    response = requests.post(
        "https://api.audiopod.ai/api/v1/media/extract",
        headers={"Authorization": f"Bearer {access_token}"},
        json={
            "url": "https://youtube.com/watch?v=dQw4w9WgXcQ",
            "format": "mp3",
            "quality": "high"
        }
    )
    
    if response.status_code == 200:
        result = response.json()
        audio_url = result["audio_url"]
        print(f"Extracted audio: {audio_url}")
    ```
  </Tab>
  <Tab title="cURL">

    urls_to_extract = [
        {"url": "https://youtube.com/watch?v=video1", "name": "intro_music"},

        {"url": "https://soundcloud.com/track1", "name": "sound_effect"}

    response = requests.post(
        "https://api.audiopod.ai/api/v1/media/extract/batch",

                "normalize_volume": True
            }
        }
    )
    
    batch_job = response.json()
    batch_id = batch_job["batch_id"]
    
    # Monitor batch progress
    def check_batch_status(batch_id):
        response = requests.get(
            f"https://api.audiopod.ai/api/v1/media/extract/batch/{batch_id}",
            headers={"Authorization": f"Bearer {access_token}"}
        )
        return response.json()
    
    # Poll until complete
    while True:
        status = check_batch_status(batch_id)
        if status["status"] == "completed":
            break
        print(f"Progress: {status['completed']}/{status['total']}")
        time.sleep(5)

## Job Management

### Monitor Extraction Progress

Track extraction jobs in real-time:

<Tabs>
  <Tab title="GET">
    ```http
    GET /media/extract/jobs/{job_id}
    Authorization: Bearer {access_token}
    ```
  </Tab>
  <Tab title="Python">
    ```python
    def monitor_extraction(job_id, callback=None):
        while True:
            response = requests.get(
                f"https://api.audiopod.ai/api/v1/media/extract/jobs/{job_id}",
                headers={"Authorization": f"Bearer {access_token}"}
            )
            
            job = response.json()
            status = job["status"]
            
            if callback:
                callback(job)
            
            if status == "completed":
                return job["result"]
            elif status == "failed":
                raise Exception(f"Extraction failed: {job['error']}")
            elif status == "processing":
                print(f"Progress: {job.get('progress', 0)}%")
            
            time.sleep(2)
    
    # Monitor with callback
    def progress_callback(job):
        print(f"Status: {job['status']} - {job.get('progress', 0)}%")
        if job.get('current_stage'):
            print(f"Stage: {job['current_stage']}")
    
    result = monitor_extraction(job_id, progress_callback)
    ```

## Best Practices

### Optimization Tips

1. **Use appropriate quality**: Don't over-extract if lower quality suffices
2. **Segment long content**: Extract specific portions instead of entire videos
3. **Batch similar requests**: Group extractions to optimize processing
4. **Cache metadata**: Store media info to avoid repeated API calls
5. **Handle failures gracefully**: Implement retry logic for transient errors

### Legal Considerations

1. **Respect copyright**: Only extract content you have rights to use
2. **Follow platform ToS**: Respect source platform terms of service
3. **Fair use guidelines**: Understand fair use provisions for your use case
4. **Attribution**: Provide proper attribution when required
5. **Commercial use**: Ensure you have rights for commercial applications

### Integration Examples

```python
# Production media extraction pipeline
class MediaExtractor:
    def __init__(self, api_key):
        self.client = AudioPodClient(api_key)
        self.quality_settings = {
            "podcast": {"format": "mp3", "bitrate": 128},
            "music": {"format": "wav", "quality": "high"},
            "voice_clone": {"format": "wav", "quality": "best"}
        }

    def extract_for_purpose(self, url, purpose, **kwargs):
        settings = self.quality_settings.get(purpose, self.quality_settings["podcast"])
        settings.update(kwargs)

        # Get media info first
        info = self.client.media.get_info(url)

        # Validate duration
        if info.duration > self.get_max_duration(purpose):
            raise ValueError(f"Content too long for {purpose}")

        # Extract with appropriate settings
        job = self.client.media.extract(url, **settings)

        # Monitor and return result
        return self.wait_for_completion(job.job_id)

    def extract_playlist_highlights(self, playlist_url, max_videos=10):
        # Extract playlist info
        playlist_info = self.client.media.get_playlist_info(playlist_url)

        # Filter and sort by popularity
        videos = sorted(
            playlist_info.videos,
            key=lambda v: v.view_count,
            reverse=True
        )[:max_videos]

        # Batch extract
        urls = [{"url": v.url, "name": v.title} for v in videos]
        batch_job = self.client.media.extract_batch(urls)

        return self.monitor_batch(batch_job.batch_id)
```
